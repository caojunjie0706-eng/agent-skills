# Document Writing Templates

## A. Pre-Writing Brief

Use before drafting when the goal is unclear.

```markdown
目标：
读者：
读者已知共识：
读者尚未形成共识：
希望推动的决策/行动：
关键问题：
核心观点：
需要的证据：
不需要展开的信息：
```

## B. 300-400 Character Story Line

```markdown
当前大环境是：[受众正在关心的业务背景、机会、风险或共识缺口]。
从当前视角看，小环境里的关键矛盾是：[相关方、分工、冲突、信息差、依赖问题]。
因此这份材料要解决的问题不是简单同步 [表层事项]，而是帮助大家形成对 [关键问题] 的共同判断。
材料会先说明 [核心判断]，再通过 [关键证据/案例/拆解] 解释为什么成立，最后明确 [决策点/原则/行动]，让各方能按同一套逻辑推进后续工作。
```

## C. Business Review Outline

```markdown
# [观点型标题：业务整体判断]

## 1. 一分钟结论
- 核心指标：
- 主要变化：
- 最大驱动：
- 最大阻力：
- 需要决策/支持：

## 2. 增长/问题拆解
- 指标公式或拆解维度：
- A：变化、原因、动作
- B：变化、原因、动作
- C：变化、原因、动作

## 3. 关键单点深挖
- 为什么选这个点：
- 看到的现象：
- 背后的系统问题：
- 后续动作：
- 需要哪些支持：

## 4. 决策点
- 选项：
- 判断标准：
- 推荐结论：
- 风险和代价：
```

```markdown
## 3. 核心观点

## 4. 论据

## 5. 共识与原则

```

## F. Review Checklist

Use this when asked to review or polish a document.

```markdown
目标是否明确：
读者是否明确：
决策点是否明确：
标题是否表达观点：
故事线是否成立：
结构是否按业务逻辑拆解：
数据是否解释了原因：
用户反馈是否结合行为验证：
是否暴露风险/冲突/依赖需求：
是否有负责人和时间：
是否有可删除的信息：
是否存在术语、缩写、空话：
是否已移除公司名、人名、内部链接、精确敏感数据和专有项目名：
```
